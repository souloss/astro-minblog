# Astro Minimax SDD 规范文档索引

本文档集用于为当前仓库建立基于规范的开发（SDD，Specification-Driven Development）基线，内容严格基于当前 monorepo 的实际结构、公开入口、关键配置与核心实现链路整理。

## 文档列表

### 1. [项目总体规范](./01-project-overview-spec.md)

说明项目定位、包划分、系统边界、总体目标、非目标、核心约束与 SDD 使用方式。

### 2. [包与目录结构规范](./02-package-and-directory-spec.md)

说明各 package / app 的职责、目录层级、关键文件作用、模块边界与依赖关系。

### 3. [核心链路与功能特性规范](./03-core-flows-and-features-spec.md)

说明 Astro 集成链路、内容链路、AI Chat / RAG 链路、通知链路、CLI / 数据链路，以及相关功能特性细节。

### 4. [需求、目标与验收规范](./04-requirements-goals-and-verification-spec.md)

说明当前项目在架构、内容、AI、通知、CLI、部署与质量保障方面的需求目标、验收要求与检验方式。

### 5. [关键文件作用规范](./05-key-file-roles-spec.md)

说明各 package 与 `apps/blog` 中关键入口文件、配置文件、共享契约文件与 API 文件的具体职责，用于补强“文件作用”这一层级的 SDD 基线。

## 使用建议

在后续进行 SDD 开发时，建议按以下顺序使用：

1. 先阅读《项目总体规范》，确认边界与目标。
2. 再阅读《包与目录结构规范》，定位改动所属包与目录。
3. 若要确认具体文件入口与职责，阅读《关键文件作用规范》。
4. 对涉及运行链路或功能行为的需求，阅读《核心链路与功能特性规范》。
5. 在落地实现、评审或验收前，对照《需求、目标与验收规范》执行检查。

## 维护原则

- 当新增 package、目录、核心链路或运行时能力时，必须同步更新对应规范。
- 当实际代码与文档冲突时，应先确认代码是否为预期变更，再决定修正文档还是修正实现。
- 规范文档描述的是“当前仓库基线”，不是未来愿景，因此不得在未实现前提前写入已支持结论。
